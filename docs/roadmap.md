# Roadmap

## Phase 0: Product Foundation

- Define product vision.
- Define v1 scope.
- Define ITIL operating model.
- Define reporting model.
- Define knowledge base strategy.
- Confirm hosting and data direction.
- Confirm working agreement, including test-driven development and delegated delivery model.

## Phase 1: Support Hub Core

- Select stack, test tooling, and CI gates.
- Create application skeleton only after initial failing tests are defined.
- Portal ticket submission.
- Agent ticket workspace.
- Manual ticket creation for phone support.
- Email-to-ticket intake.
- Ticket statuses, categories, priority, assignment, and comments.
- Basic SLA tracking.
- Basic reporting.

## Phase 2: Knowledge Hub

- Article authoring and publishing.
- Customer-facing knowledge search.
- Internal-only articles.
- Ticket-to-article linking.
- Article helpfulness feedback.
- Knowledge gap reporting.

## Phase 3: Chat and Bot Escalation

- Customer chat entry point.
- Knowledge-grounded answer suggestions.
- Escalation from chat to ticket.
- Conversation transcript attached to ticket.
- Bot deflection reporting.

## Phase 4: Operational Maturity

- Problem records for recurring incidents.
- Change records for controlled operational updates.
- Service catalog for common requests.
- Customer satisfaction capture.
- Enhanced dashboards for agents, leads, and executives.

## Phase 5: Data Platform and Glass Hub

- Export operational data to Microsoft Fabric.
- Build Fabric lakehouse or warehouse model.
- Create Power BI semantic model.
- Build executive support dashboard.
- Add cross-system reporting as Customer Hub and other hubs come online.
